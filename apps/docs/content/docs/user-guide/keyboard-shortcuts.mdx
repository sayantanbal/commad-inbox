Command Inbox follows **Superhuman-style** keyboard conventions. Shortcuts are disabled when modals (composer, palette, search) are open.

**Mod** = `⌘` on Mac, `Ctrl` on Windows/Linux.

## Global shortcuts

Available from most inbox views.

| Key | Action |
|-----|--------|
| **`/`** | Open semantic search |
| **`?`** (`Shift+/`) | Open shortcut cheat sheet |
| **`Mod+K`** | Open command palette |
| **`Mod+Shift+F`** | Open advanced Gmail search |

## List shortcuts

When focus is on the thread list (no composer open).

| Key | Action |
|-----|--------|
| **`J`** | Next thread |
| **`K`** | Previous thread |
| **`X`** | Toggle multi-select |

## Thread shortcuts

Requires a selected thread.

| Key | Action |
|-----|--------|
| **`E`** | Archive (move to Done lane) |
| **`R`** | Reply — opens composer |
| **`M`** | Schedule or reschedule meeting |
| **`S`** | Snooze thread |

## Composer shortcuts

When the reply composer is open.

| Key | Action |
|-----|--------|
| **`Mod+Enter`** | Send email |
| **`Escape`** | Close composer |

## Command palette only

These actions appear in **`Mod+K`** but have no dedicated key binding:

| Action | Description |
|--------|-------------|
| Cancel linked meeting | Removes calendar event linked to current thread |

## Shortcut gating rules

The app disables shortcuts intelligently:

1. **Single-key shortcuts** (`J`, `K`, `E`, etc.) are suppressed when any overlay is open — search, palette, snooze picker, availability panel, or cheat sheet
2. **Thread actions** require a selected thread (`requiresThread`)
3. **Composer shortcuts** only work when the composer is open; other thread shortcuts are blocked while composing

## Mobile

On mobile, use **tabs** at the bottom (Inbox / Thread / Calendar) and **swipe gestures** on the thread list. Keyboard shortcuts apply when using an external keyboard or desktop browser.
